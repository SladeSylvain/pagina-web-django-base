# Proyecto de Prueba: Página Web Molde

Este proyecto consiste en una página web de prueba que funciona como una plantilla o molde base, diseñada específicamente para ser reutilizada, adaptada y escalada en futuros desarrollos de manera rápida y eficiente.

---

## 🛠️ Tecnologías Utilizadas

En este proyecto se implementaron las siguientes tecnologías para asegurar una estructura sólida y un diseño limpio:

*   **HTML5:** Creación de la estructura semántica y la arquitectura base del sitio.
*   **CSS3:** Diseño visual, estilos personalizados y layouts responsivos.
*   **JavaScript (Vanilla):** Implementación de la lógica interactiva del lado del cliente.
*   **Visual Studio Code:** Entorno de desarrollo principal, utilizando extensiones clave para la optimización del flujo de trabajo (como Live Server).

---

## 👣 Paso a Paso del Desarrollo

Para construir esta plantilla, se siguió un proceso ordenado en VS Code:

1.  **Inicialización del Proyecto:** Creación de la carpeta raíz del proyecto y la estructura de archivos esencial (`index.html`, `style.css` y `app.js`).
2.  **Maquetación Semántica:** Construcción del esqueleto en `index.html` utilizando etiquetas semánticas para garantizar un buen SEO y accesibilidad.
3.  **Estilizado Base y Layout:** Creación de las reglas CSS para definir la paleta de colores, tipografías y el comportamiento responsivo del diseño.
4.  **Inyección de Interactividad:** Vinculación del archivo JavaScript para manejar los eventos dinámicos esenciales de la página.
5.  **Pruebas en Tiempo Real:** Uso de servidores locales para previsualizar los cambios instantáneamente y depurar errores de renderizado.

---

## 🧠 Retrospectiva y Aprendizaje

Un componente clave de este trabajo fue el proceso de ensayo y error, lo que permitió identificar áreas de mejora en la escritura de código.

### ❌ En qué me equivoqué
*   **Errores de Sintaxis Iniciales:** Al escribir la estructura o las reglas de estilo, algunos pequeños descuidos tipográficos o de cierre de etiquetas afectaron temporalmente el comportamiento del diseño.
*   **Manejo de Rutas:** Hubo confusiones momentáneas al enlazar los archivos externos (CSS/JS) debido a la declaración de rutas relativas.

### 📈 En qué aspecto mejoré
*   **Depuración de Código (Debugging):** Mayor agilidad para leer los mensajes de error en la consola del navegador y localizarlos directamente en VS Code.
*   **Buenas Prácticas:** Incremento en la atención al detalle, asegurando un código más limpio, identado y ordenado desde el principio.
*   **Autonomía Técnica:** Mejor capacidad para identificar la causa raíz de un fallo visual o de lógica y corregirlo de forma independiente.
